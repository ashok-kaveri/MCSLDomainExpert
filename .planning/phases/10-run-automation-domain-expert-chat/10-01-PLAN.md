<feature>
  <name>pipeline/test_runner.py — Playwright spec runner and result parser</name>
  <files>pipeline/test_runner.py, tests/test_run_automation.py</files>
  <behavior>
    - test_run01_enumerate_specs: enumerate_specs(fake_repo_path) where fake_repo_path/tests/ has two nested spec files returns dict with folder keys and list of relative paths as values; files not ending in .spec.ts are excluded
    - test_run01_test_run_result_dataclass: TestRunResult() has fields total, passed, failed, skipped, duration_ms (all int default 0), specs (list default []), error (str default ""); SpecResult has file, title, status, duration_ms fields
    - test_run01_parse_playwright_json: given a fixture dict with one suite containing one spec with status="passed" and duration=1500, parse_playwright_json returns list with one entry having status="passed" and duration_ms=1500
    - test_run01_run_release_tests_calls_subprocess: mock subprocess.run to return a fake process (returncode=0); mock tempfile and Path.read_text to return valid JSON; run_release_tests("fake_path", ["tests/foo.spec.ts"]) returns a TestRunResult with no error field set
  </behavior>
  <implementation>
    Implement pipeline/test_runner.py with:

    1. SpecResult dataclass: file: str, title: str, status: str ("passed"|"failed"|"timedOut"|"skipped"), duration_ms: int

    2. TestRunResult dataclass: specs: list[SpecResult] = field(default_factory=list), total: int = 0, passed: int = 0, failed: int = 0, skipped: int = 0, duration_ms: int = 0, error: str = ""

    3. enumerate_specs(repo_path: str) -> dict[str, list[str]]:
       - Walk Path(repo_path) / "tests" with rglob("*.spec.ts")
       - Group by top-level subfolder: parts = spec.relative_to(root).parts; folder = parts[0] if len(parts) > 1 else "root"
       - Return rel path as str(spec.relative_to(Path(repo_path)))
       - Return dict sorted alphabetically; each folder's list sorted
       - If path does not exist, return {}

    4. _parse_suite(suite: dict, results: list) -> None:
       - file_name = suite.get("file", suite.get("title", ""))
       - For each spec in suite.get("specs", []), for each test in spec.get("tests", []), take first result
       - Append {"file": file_name, "title": spec title, "status": status, "duration_ms": duration}
       - Recurse into suite.get("suites", [])

    5. parse_playwright_json(data: dict) -> list[dict]:
       - Call _parse_suite for each top-level suite
       - Return flat list of result dicts

    6. run_release_tests(repo_path: str, spec_files: list[str], project: str = "Google Chrome") -> TestRunResult:
       - Use tempfile.NamedTemporaryFile(suffix=".json", delete=False) to get json_path
       - env = {**os.environ, "PLAYWRIGHT_JSON_OUTPUT_FILE": json_path}
       - cmd = ["npx", "playwright", "test", "--reporter=json", "--project", project, *spec_files]
       - subprocess.run(cmd, cwd=repo_path, env=env, capture_output=True, text=True, timeout=600)
       - Read json_path if it exists, parse with parse_playwright_json
       - Build TestRunResult: count passed/failed/skipped from status, sum duration_ms
       - Clean up json_path with os.unlink (ignore errors)
       - On any exception, return TestRunResult(error=str(e))
       - never raises — all errors go into TestRunResult.error

    No new packages needed — subprocess, tempfile, pathlib, dataclasses, json, os are all stdlib.
  </implementation>
</feature>
